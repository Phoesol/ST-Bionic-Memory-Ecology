# Product

## Register

product

## Users
SillyTavern users who run Codex-assisted roleplay workflows and need memory, briefing, world state, and workflow controls in one panel.

## Product Purpose
Codex-Momery is the unified control surface for memory graph work, scene briefing, world evolution, and Codex auxiliary tasks. Success means the user opens one floating entry and switches pages without managing multiple overlapping extension buttons.

## Brand Personality
Dense, calm, technical.

## Anti-references
Avoid separate floating panels, decorative dashboards, duplicated API configuration, and UI that visually conflicts with the ST-BME panel.

## Design Principles
- Keep one entry point: the memory graph floating button owns the combined workflow.
- Preserve working module boundaries while presenting one consistent interface.
- Prefer status-rich controls over explanatory text.
- Keep Codex-owned tasks file-backed and readable by the assistant.

## Accessibility & Inclusion
Use the existing ST-BME theme tokens, keyboard-focusable buttons, restrained motion, and mobile-safe navigation.
